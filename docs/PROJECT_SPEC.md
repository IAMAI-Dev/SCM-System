# 供应链管理系统项目规范

## 项目定位

本项目是面向课程实验与桌面业务演示的供应链管理系统。当前主工程采用
PySide6 与 MySQL，旧版 `scm_gui.py` 仅作对照。系统应保持企业桌面软件
的稳定感、数据密度和短操作路径。

## 技术与模块

- GUI：PySide6。
- 数据库：MySQL 8.x、mysql-connector-python 连接池。
- 图表：Matplotlib QtAgg。
- 导出：pandas 与 openpyxl。
- 模块：登录、欢迎页、仪表盘、订单、库存、客户、供应商、采购、经营
  分析、性能监控、用户权限和审计日志。

数据库名不属于代码常量。示例配置使用 `experiment2026`，实际环境由
`config.ini` 或 `SCM_DB_*` 环境变量决定。

## 分层边界

| 层级 | 职责 | 禁止事项 |
| --- | --- | --- |
| `ui/` | 控件、布局、交互、显示状态 | 直接编写业务 SQL |
| `service/` | 权限、规则、格式转换、事务编排 | 创建 Qt 控件 |
| `dao/` | 参数化 SQL、存储过程与数据读写 | 决定界面文案 |
| `core/` | 配置、连接池、事务、迁移和基础异常 | 依赖具体业务页面 |
| `sql/` | 有序迁移、视图、索引、过程、触发器 | 保存真实密码 |
| `utils/` | 导出等跨模块通用能力 | 绕过 Service 写业务数据 |

写操作必须先在 Service 层校验权限，再通过事务调用 DAO。订单、补货和
权限变更的业务数据与审计日志应在同一事务内提交或回滚。

## 权限规范

- `admin`：全部模块和操作。
- `manager`：可查看跨部门业务和经营分析，只能修改所属部门业务。
- `staff`：只进入所属部门模块，并按权限执行查看或新增。
- 页面入口控制只用于改善体验，Service 层必须再次校验真实权限。
- 登录、权限修改、状态更新和关键写操作必须记录审计日志。

## UI 与异步规范

界面使用工业调度控制台配色：

- 侧边栏 `#1f2428`
- 顶部工具栏 `#f8f5ee`
- 主背景 `#f3f1ec`
- 内容与图表 `#fffdf8`
- 主强调色 `#b56b2a`
- 次强调色 `#2f6f73`

表格是业务页面的核心区域。分析页应遵守以下约束：

1. 页面框架先显示，数据库查询放入 `QThread` Worker。
2. 加载状态统一为 `idle/loading/loaded/error`。
3. Worker 只查询数据，不创建或修改 Qt 控件。
4. Matplotlib 画布在主线程创建和绘制。
5. 隐藏画布必须先切换为可见，再在下一轮事件循环执行最终绘制。
6. 动态包含画布的卡片不得长期保留 `QGraphicsEffect` 缓存。
7. 刷新失败时保留上一版有效数据，并在固定状态区域提示。
8. 页面需在 `1320x840` 和最小窗口 `980x620` 下无重叠与横向溢出。

## 数据库规范

- 禁止硬编码数据库账号和密码。
- 所有外部输入必须通过 `%s` 参数化。
- 游标和连接必须由上下文管理器释放。
- 普通查询使用 `cursor()`，写操作使用 `transaction()`。
- 数据库对象通过 `scm_schema_migrations` 版本化管理。
- 已发布迁移不修改语义；变更应新增递增编号 SQL，并登记到
  `MIGRATION_SCRIPTS`。
- 索引变更需说明目标查询，并在真实数据量下验证执行计划和耗时。

## 代码与协作

- Python 遵循 PEP 8；注释只解释不明显的意图。
- 导入顺序为标准库、第三方库、本地模块。
- 不提交 `.idea`、虚拟环境、图表缓存、调试截图和真实 `config.ini`。
- 每次功能修改至少运行编译检查和 `tools/smoke_check.py`。
- GUI 修改需要首次进入、刷新、切页返回和最小窗口截图验证。
- 提交信息使用 `<type>: <简短说明>`，常用类型包括 `feat`、`fix`、
  `docs`、`refactor`、`sql` 和 `chore`。
