# 供应链管理系统新版项目规范

## 一、项目定位

本项目是基于 PySide6 与 MySQL 的供应链管理桌面系统。团队成员应以
克隆后的项目根目录为主工程目录，外层需求材料和旧版项目仅作为参考。

系统应体现真实企业桌面软件的稳定感、数据密度和短操作路径，
避免营销页式布局、深蓝科技渐变、玻璃拟态和强发光效果。

## 二、技术栈

- GUI 框架：PySide6。
- 数据库：MySQL，默认连接 `experiment2026` 数据库。
- 数据库驱动：mysql-connector-python。
- 图表：matplotlib，嵌入 PySide6 页面。
- 架构：UI、Service、DAO、Core 分层。

## 三、分层规范

- `ui/`：只负责界面、交互、展示和信号连接，不允许写 SQL。
- `service/`：负责业务规则、权限判断、事务编排和数据格式整理。
- `dao/`：负责参数化 SQL、存储过程调用和数据库对象访问。
- `core/`：负责配置读取、数据库连接、异常和通用基础能力。
- `sql/`：保存应用辅助表、视图、索引、存储过程、触发器与种子数据。
- `utils/`：保存导出、格式化、验证等通用工具。

## 四、界面规范

界面使用“工业调度控制台”风格：

- 侧边栏背景：`#1f2428`。
- 顶部工具栏背景：`#f8f5ee`。
- 主内容背景：`#f3f1ec`。
- 卡片与表格背景：`#fffdf8`。
- 主强调色：`#b56b2a`。
- 次强调色：`#2f6f73`。
- 成功、警告、危险状态色分别使用 `#4f7d5a`、`#c9821f`、
  `#b84536`。

表格是核心视觉区域。优先使用 `QTableView` 与
`QAbstractTableModel`，支持筛选、刷新、分页、状态标签和批量操作。

## 五、数据库规范

- 禁止在代码中硬编码数据库账号、密码和真实私有配置。
- 提交 `config.ini.example`，真实 `config.ini` 加入 `.gitignore`。
- 所有 SQL 必须使用 `%s` 占位符参数化，禁止字符串拼接。
- DAO 获取连接后必须关闭游标并释放连接。
- 下单、库存扣减、补货等写操作必须使用事务。
- 保留数据库课程亮点：视图、索引、存储过程、触发器、事务、
  审计日志。

## 六、代码与协作规范

- Python 代码遵循 PEP 8，行宽不超过 79 字符。
- 注释、docstring、文档和提交信息统一使用中文。
- import 分为标准库、第三方库、本地模块三组。
- UI 耗时查询必须放入后台线程，避免阻塞主线程。
- 每次阶段性修改必须提交，禁止一次提交过多代码。
- 提交格式：

```text
<type>: <中文标题>
```

常用 `type` 包括 `feat`、`fix`、`docs`、`style`、`refactor`、
`sql`、`chore`。
