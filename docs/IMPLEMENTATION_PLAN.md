# 供应链管理系统实施状态与维护计划

## 当前状态

旧版 Tkinter 单文件系统已经迭代为 PySide6 + MySQL 分层桌面系统，主入口
为 `main.py`。当前阶段的架构重构、核心业务补全、数据库迁移和分析页
性能优化已经完成。

## 已完成阶段

1. **工程与配置**
   - 建立 `core/dao/service/ui/sql/utils/tools` 分层目录。
   - 支持 `config.ini` 与环境变量覆盖，不提交真实密码。

2. **数据库基础设施**
   - 实现连接池、只读游标和事务上下文。
   - 建立应用用户、日志、订单审计和补货订单表。
   - 建立视图、存储过程、触发器和性能索引。
   - 使用 `scm_schema_migrations` 只执行待处理迁移。

3. **认证与业务模块**
   - 完成登录、部门角色权限和 Service 层二次校验。
   - 完成仪表盘、订单、库存、客户、供应商、采购、用户和审计模块。
   - 增加经营分析、性能监控和 Excel 导出。

4. **GUI 与性能**
   - 主窗口和页面按需创建，登录后先显示轻量欢迎页。
   - 仪表盘与分析查询改为后台线程。
   - Matplotlib 在登录期间预热，图表使用局部加载状态与单次绘制。
   - 完成供应商图例、经营分析标签和首次进入画布刷新修复。

5. **验证与文档**
   - 建立编译与静态冒烟检查。
   - 完成真实 MySQL 联调和桌面/最小窗口截图检查。
   - 更新 README、项目规范、运维说明、变更记录和历史任务状态。

## 后续维护流程

每次迭代按以下顺序进行：

1. 阅读 `README.md`、`docs/PROJECT_SPEC.md` 和相关 Service/DAO。
2. 检查工作树，保留团队成员未提交的修改。
3. 数据库变更新增编号迁移，不重写已发布迁移。
4. 耗时查询使用后台 Worker；Qt 与 Matplotlib 操作留在主线程。
5. 执行编译、冒烟、数据库和 GUI 首次进入测试。
6. 更新 README、Changelog 与相关运维说明后再提交。

## 验收基线

- 普通启动只做轻量迁移版本检查。
- 登录成功后主窗口不等待仪表盘业务查询。
- 分析页面首次进入即可显示图表，无需切页触发重绘。
- 加载和刷新期间导航、拖动、缩放保持响应。
- 写操作权限、事务和审计保持一致。
- UI 层无业务 SQL，仓库无真实数据库凭据。
